## 🔴 第二轮 — 核心体验提升（高优先级）

### 2.1 首页灵感输入（Seko 核心交互）

**现状**: 仪表盘只有导航按钮，用户不知道从哪开始
**目标**: 在仪表盘顶部增加「灵感输入框」，用户输入一句话即可启动全流程

```
┌─────────────────────────────────────────────────┐
│  🎬 AI 短剧工作台                                │
│  从剧本到成片，AI 全流程驱动                       │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │ 输入你的剧情灵感，例如：                    │    │
│  │ "林夏独自在家等顾辰来过生日..."             │    │
│  │                              [🚀 生成]   │    │
│  └─────────────────────────────────────────┘    │
│  [📝 分镜] [🎬 管线] [🤖 AI 生成]              │
└─────────────────────────────────────────────────┘
```

**改动范围**:
- `app.js`: `loadDashboard()` 增加灵感输入框 + 生成按钮
- 点击「生成」→ 调用 `/llm/storyboard` → 跳转分镜表
- 支持「快速模式」（一键生成）和「高级模式」（可设置集数/时长/角色）

### 2.2 角色/场景图片支持

**现状**: 角色卡片只显示文字描述首字母，无实际图片
**目标**: 支持上传/生成角色定妆照，卡片显示真实缩略图

**改动范围**:
- 后端: 角色 CRUD 增加 `reference_images` 字段存储
- 前端: 编辑面板增加图片上传区 + 拖拽上传
- 卡片: 优先显示定妆照，无图时回退到首字母占位
- 场景同理: 支持场景参考图上传

### 2.3 分镜表缩略图预览

**现状**: 时间轴视图中缩略图需要逐个异步加载，首次显示为 🎬 图标
**目标**: 首帧生成后自动显示缩略图，支持点击放大

**改动范围**:
- 时间轴卡片的 `timeline-thumb` 区域显示 frame.png
- 点击缩略图 → 资源预览浮层（已实现，需优化加载体验）
- 增加骨架屏/loading 占位

### 2.4 生产管线步骤状态联动

**现状**: 流程图步骤状态是静态的
**目标**: 执行任务时，流程图实时高亮当前步骤

**改动范围**:
- `runOne()` / `batchRun()` 执行时，更新 `pipeline-step` 的 class
- `active` → 脉冲动画 → `done` / `fail`
- 批量完成后，显示各步骤完成率

### 2.5 编辑面板增强

**现状**: 编辑面板字段平铺，长文本输入体验差
**目标**: 参考 Seko 的编辑体验

**改动范围**:
- 动作/台词输入框增加字数统计
- 角色/场景字段改为下拉选择（从已有配置中选）
- 增加「预览」按钮，直接在编辑面板中预览当前镜头资源
- 保存后自动跳到下一个镜头（连续编辑模式）

## 🟡 第三轮 — 效率与专业感（中优先级）

### 3.1 分镜表拖拽排序

**现状**: 镜头顺序固定，无法调整
**目标**: 支持拖拽调整镜头顺序

**改动范围**:
- 引入 SortableJS（CDN，~10KB）
- 时间轴视图和表格视图都支持拖拽
- 拖拽后自动保存

### 3.2 批量导入/导出

**现状**: 分镜只能手动添加或 AI 生成
**目标**: 支持 CSV/JSON 导入导出

**改动范围**:
- 分镜表工具栏增加「导入」「导出」按钮
- 导入: 解析 CSV → 校验 → 合并/覆盖
- 导出: 下载当前集分镜为 CSV

### 3.3 角色/场景与分镜联动

**现状**: 角色/场景配置与分镜表独立，编辑角色后需手动更新分镜
**目标**: 角色/场景变更自动同步到分镜表

**改动范围**:
- 编辑角色时，提示「N 个分镜引用了此角色，是否同步更新？」
- 删除角色时，提示引用数量
- 角色卡片显示「被引用次数」

### 3.4 配置预设模板

**现状**: 设置页需要手动填写所有字段
**目标**: 提供常用配置预设

**改动范围**:
- 设置页增加「快速配置」区
- 预设: 「本地 ComfyUI」「云端 SiliconFlow + MIMO」「Ollama 本地」
- 一键应用预设，自动填充所有字段

### 3.5 成片预览区

**现状**: 后期合成完成后，用户需要去文件系统找 final.mp4
**目标**: 管线页面底部增加「成片预览」区

**改动范围**:
- 检测 `output/e{ep}/episode_{ep}_final.mp4` 是否存在
- 存在时显示视频播放器 + 下载按钮
- 不存在时显示「尚未生成成片」

## 🟢 第四轮 — 高级功能（低优先级）

### 4.1 对话式编辑（Seko 核心特性）

**现状**: 所有编辑都通过表单
**目标**: 增加「对话式」编辑入口，用自然语言修改分镜

**实现思路**:
- 管线页面增加「💬 对话编辑」浮窗
- 用户输入: "把第 3 个镜头的台词改成..."
- 后端调用 LLM 解析意图 → 执行对应操作
- 返回操作结果 + 更新后的分镜

### 4.2 主体库管理（Seko 特性）

**现状**: 角色定妆照散落在各项目目录
**目标**: 全局主体库，跨项目复用角色

**改动范围**:
- 新增「主体库」页面（侧边栏新增入口）
- 全局存储: `shared_assets/characters/`
- 角色卡片支持「复制到当前项目」

### 4.3 多剧集管理

**现状**: 支持多集但 UI 上没有集数总览
**目标**: 项目级集数管理面板

**改动范围**:
- 项目管理页面增加「集数列表」
- 每集显示: 镜头数 / 时长 / 状态（未开始/进行中/已完成）
- 支持批量操作: 全集生成 / 全集导出

### 4.4 实时协作指示

**现状**: 单用户使用
**目标**: 显示当前是否有任务在执行

**改动范围**:
- 侧边栏底部显示「🔧 Worker 运行中 · 3 个任务」
- 导航项上显示角标（如有进行中的任务）

## 📐 设计规范

### 色彩系统（已有，保持一致）

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | `#6c8cff` | 按钮、链接、高亮 |
| 成功 | `#34d399` | 完成状态 |
| 错误 | `#f87171` | 失败状态 |
| 警告 | `#fbbf24` | 进行中/待处理 |
| 辅助 | `#a78bfa` | AI 相关功能 |

### 组件规范

| 组件 | 用途 | 优先级 |
|------|------|--------|
| `.entity-card` | 角色/场景卡片 | 高 |
| `.timeline-item` | 分镜时间轴项 | 高 |
| `.pipeline-step` | 管线流程步骤 | 高 |
| `.stat-card` | 统计数据卡片 | 中 |
| `.quick-entry` | 快速入口卡片 | 中 |
| `.empty-state` | 空状态引导 | 中 |
| `.view-toggle` | 视图切换按钮组 | 低 |

### 交互规范

| 场景 | 行为 |
|------|------|
| 删除操作 | `modalConfirm` 二次确认 |
| 文本输入 | `modalPrompt` 自定义弹窗 |
| 长文本 | 截断 + tooltip（30 字） |
| 加载中 | 按钮禁用 + `⏳ ...` |
| 操作成功 | 绿色 toast 3.5s |
| 操作失败 | 红色 toast 3.5s |
| 未保存离开 | `modalConfirm` 确认 |
